# สรุประบบงานเอกสาร (Documents System)

## ภาพรวมระบบ

ระบบงานเอกสารเป็นระบบจัดการเอกสารที่ครอบคลุมการอัปโหลด จัดเก็บ จัดหมวดหมู่ อนุมัติ ติดตามวันหมดอายุ จัดการเวอร์ชัน กำหนดสิทธิ์ และรายงานต่างๆ สำหรับเอกสารทั้งหมดในองค์กร

---

## ฟังก์ชันหลักทั้งหมด

### 1. Dashboard (ภาพรวม)

**หน้าที่:** แสดงภาพรวมข้อมูลเอกสารทั้งหมดแบบเรียลไทม์

**ข้อมูลที่แสดง:**
- เอกสารทั้งหมด (Active)
- ใกล้หมดอายุ (ภายใน 30 วัน)
- รออนุมัติ (Pending)
- ค่าต่ออายุ (ภายใน 30 วัน)
- การดำเนินการด่วน (อัปโหลดเอกสาร, สร้างหมวดหมู่, ตรวจสอบหมดอายุ, อนุมัติเอกสาร)
- เอกสารใกล้หมดอายุ (รายการ 5 รายการล่าสุด พร้อมจำนวนวันที่เหลือ)
- การแจ้งเตือนล่าสุด (5 รายการล่าสุด: Expiring, Expired, Approval, Update)
- เอกสารแยกตามหมวดหมู่ (จำนวนเอกสารแต่ละหมวดหมู่)
- ปฏิทินเอกสารใกล้หมดอายุ (แสดงปฏิทินพร้อมเอกสารที่ใกล้หมดอายุในแต่ละวัน, สีแยกตามความเร่งด่วน, Export เป็น iCal/Google Calendar)

**Flow การทำงาน:**
1. ระบบดึงข้อมูลจาก Documents, Categories, Approvals, Notifications
2. คำนวณสถิติแบบ Real-time
3. แสดงข้อมูลในรูปแบบ Summary Cards, Recent Activity Lists, และ Calendar View

---

### 2. AllDocuments (เอกสารทั้งหมด)

**หน้าที่:** จัดการเอกสารทั้งหมดในระบบ

**ข้อมูลที่จัดการ:**
- เลขที่เอกสาร, ชื่อเอกสาร, คำอธิบาย
- หมวดหมู่
- ประเภทไฟล์ (PDF, JPG, PNG, DOC, DOCX, XLS, XLSX, OTHER)
- ขนาดไฟล์
- เวอร์ชัน
- วันหมดอายุ (ถ้ามี)
- ค่าต่ออายุ (ถ้ามี)
- สถานะ (Active, Archived, Draft, Pending)
- ระดับความลับ (Public, Internal, Confidential)
- ผู้สร้าง, วันที่สร้าง
- ผู้แก้ไขล่าสุด, วันที่แก้ไขล่าสุด
- ผู้อนุมัติ, วันที่อนุมัติ
- Tags
- เชื่อมโยงกับโมดูลอื่น (HR, Fund, Accounting) และ Transaction ID
- Metadata (ข้อมูลเพิ่มเติม เช่น เลขที่บิล, ยอดรวม)

**ฟังก์ชันหลัก:**
- อัปโหลดเอกสารใหม่ (กรอกข้อมูล, เลือกหมวดหมู่, อัปโหลดไฟล์, กำหนดระดับความลับ, วันหมดอายุ, Tags)
- ดูรายละเอียดเอกสาร (ข้อมูลทั้งหมด, Tags, การเชื่อมโยง, Metadata)
- ดาวน์โหลดเอกสาร
- แก้ไขเอกสาร
- ค้นหาและกรอง (ตามชื่อ, เลขที่, คำอธิบาย, Tags, หมวดหมู่, สถานะ, ระดับความลับ)

**Flow การทำงาน:**
1. อัปโหลดเอกสาร:
   - กรอกชื่อเอกสาร, เลือกหมวดหมู่, คำอธิบาย
   - อัปโหลดไฟล์ (รองรับ PDF, JPG, PNG, DOC, DOCX, XLS, XLSX)
   - กำหนดระดับความลับ, วันหมดอายุ (ถ้ามี), Tags
   - บันทึก → สถานะเป็น Draft หรือ Pending (รออนุมัติ)
2. ดูรายละเอียด:
   - คลิก "ดูรายละเอียด" → แสดงข้อมูลทั้งหมด, Tags, การเชื่อมโยง, Metadata
   - สามารถเชื่อมโยงกับโมดูลอื่น (HR, Fund, Accounting)
3. แก้ไข/ดาวน์โหลด:
   - คลิก "แก้ไข" หรือ "ดาวน์โหลด" → ดำเนินการตามต้องการ

**เงื่อนไขสำคัญ:**
- รองรับไฟล์: PDF, JPG, PNG, DOC, DOCX, XLS, XLSX
- ระดับความลับ: Public (สาธารณะ), Internal (ภายใน), Confidential (ลับ)
- สามารถเชื่อมโยงกับโมดูลอื่นและ Transaction ID

---

### 3. Categories (หมวดหมู่)

**หน้าที่:** จัดการหมวดหมู่เอกสาร

**ข้อมูลที่จัดการ:**
- ชื่อหมวดหมู่
- คำอธิบาย
- สี (blue, green, purple, orange, red, yellow, gray)
- จำนวนเอกสารในหมวดหมู่

**ฟังก์ชันหลัก:**
- เพิ่มหมวดหมู่ใหม่ (กรอกชื่อ, คำอธิบาย, เลือกสี)
- แก้ไขหมวดหมู่
- ลบหมวดหมู่ (ต้องไม่มีเอกสารในหมวดหมู่)
- ดูจำนวนเอกสารในแต่ละหมวดหมู่

**Flow การทำงาน:**
1. เพิ่มหมวดหมู่: กรอกชื่อ → คำอธิบาย (ถ้ามี) → เลือกสี → บันทึก
2. แก้ไข: คลิก "แก้ไข" → แก้ไขข้อมูล → บันทึก
3. ลบ: คลิก "ลบ" → ยืนยัน (ต้องไม่มีเอกสารในหมวดหมู่)

**เงื่อนไขสำคัญ:**
- ไม่สามารถลบหมวดหมู่ที่มีเอกสารอยู่ได้

---

### 4. Approvals (การอนุมัติ)

**หน้าที่:** จัดการการอนุมัติเอกสารตาม Workflow

**ข้อมูลที่จัดการ:**
- เอกสารที่รออนุมัติ
- ขั้นตอนการอนุมัติ (Step 1, 2, 3, ...)
- ผู้อนุมัติในแต่ละขั้นตอน
- สถานะ (Pending, Approved, Rejected, Cancelled)
- วันที่สร้าง, วันที่ลงลายเซ็น
- ความคิดเห็น
- ลายเซ็นอิเล็กทรอนิกส์ (E-Signature)

**ฟังก์ชันหลัก:**
- ดูรายการเอกสารที่รออนุมัติ (จัดกลุ่มตามเอกสาร, แสดงขั้นตอนการอนุมัติ)
- อนุมัติเอกสาร (พร้อมหรือไม่พร้อมลายเซ็น)
- ไม่อนุมัติเอกสาร (ระบุเหตุผล)
- ลงลายเซ็นอิเล็กทรอนิกส์ (วาดลายเซ็น หรืออัปโหลดรูปภาพ)
- ค้นหาและกรอง (ตามสถานะ)

**Flow การทำงาน:**
1. ดูรายการ: แสดงเอกสารที่รออนุมัติ พร้อมขั้นตอนการอนุมัติทั้งหมด
2. อนุมัติ:
   - คลิก "อนุมัติ" → เลือกลงลายเซ็น (ถ้าต้องการ) → อนุมัติ
   - ลงลายเซ็น: เลือก "วาดลายเซ็น" หรือ "อัปโหลดรูปภาพ" → บันทึกลายเซ็น → อนุมัติ
3. ไม่อนุมัติ:
   - คลิก "ไม่อนุมัติ" → ระบุเหตุผล → บันทึก

**เงื่อนไขสำคัญ:**
- การอนุมัติเป็นแบบ Multi-step Workflow
- สามารถลงลายเซ็นอิเล็กทรอนิกส์ได้ (วาดหรืออัปโหลด)
- ระบบติดตามขั้นตอนการอนุมัติอัตโนมัติ

---

### 5. Expiring (ใกล้หมดอายุ)

**หน้าที่:** ติดตามเอกสารที่ใกล้หมดอายุและต้องต่ออายุ

**ข้อมูลที่แสดง:**
- เอกสารที่หมดอายุแล้ว
- เอกสารที่หมดอายุใน 7 วัน
- เอกสารที่หมดอายุใน 15 วัน
- เอกสารที่หมดอายุใน 30 วัน
- ค่าต่ออายุรวม (จากรายการที่แสดง)
- รายละเอียดเอกสาร (เลขที่, ชื่อ, หมวดหมู่, วันหมดอายุ, จำนวนวันที่เหลือ, ค่าต่ออายุ)

**ฟังก์ชันหลัก:**
- ดูรายการเอกสารใกล้หมดอายุ (ตาราง หรือ ปฏิทิน)
- กรองตามช่วงเวลา (หมดอายุแล้ว, ≤ 7 วัน, 8-15 วัน, 16-30 วัน)
- ดูรายละเอียดเอกสาร
- สลับมุมมอง (ตาราง / ปฏิทิน)

**Flow การทำงาน:**
1. ดูรายการ: แสดงเอกสารที่ใกล้หมดอายุ (≤ 30 วัน) พร้อมจำนวนวันที่เหลือ
2. กรอง: เลือกช่วงเวลา → แสดงเฉพาะเอกสารในช่วงเวลานั้น
3. ดูปฏิทิน: สลับเป็นมุมมองปฏิทิน → แสดงเอกสารที่ใกล้หมดอายุในแต่ละวัน (สีแยกตามความเร่งด่วน)
4. ดูรายละเอียด: คลิก "ดูรายละเอียด" → แสดงข้อมูลเอกสารและคำเตือน

**เงื่อนไขสำคัญ:**
- ระบบคำนวณจำนวนวันที่เหลืออัตโนมัติ
- สีแยกตามความเร่งด่วน: แดง (≤ 7 วัน), ส้ม (8-15 วัน), เหลือง (16-30 วัน)

### 6. AuditTrail (Audit Trail)

**หน้าที่:** ติดตามประวัติการเปลี่ยนแปลงเอกสารทั้งหมด

**ข้อมูลที่ติดตาม:**
- เวลา (Timestamp)
- การกระทำ (Created, Updated, Deleted, Viewed, Downloaded, Approved, Rejected)
- เอกสาร (Document ID, ชื่อเอกสาร)
- ผู้ทำ (User ID, ชื่อผู้ใช้)
- รายละเอียด (Details)
- IP Address
- User Agent

**ฟังก์ชันหลัก:**
- ดูประวัติการเปลี่ยนแปลงทั้งหมด (เรียงตามเวลาล่าสุด)
- ค้นหาและกรอง (ตามการกระทำ, ผู้ใช้, วันที่)
- ดูรายละเอียด Audit Trail
- ส่งออกรายงาน Audit Trail

**Flow การทำงาน:**
1. ดูประวัติ: แสดงตารางประวัติการเปลี่ยนแปลงทั้งหมด
2. กรอง: เลือกการกระทำ, ผู้ใช้, ช่วงวันที่ → แสดงเฉพาะรายการที่ตรงกับเงื่อนไข
3. ดูรายละเอียด: คลิก "ดูรายละเอียด" → แสดงข้อมูลทั้งหมด (เวลา, การกระทำ, ผู้ทำ, IP Address, User Agent, รายละเอียด)
4. ส่งออก: คลิก "ส่งออกรายงาน" → ส่งออกเป็น Excel/PDF

**เงื่อนไขสำคัญ:**
- บันทึกทุกการกระทำอัตโนมัติ
- เก็บข้อมูล IP Address และ User Agent สำหรับการตรวจสอบ

### 7. Versions (เวอร์ชัน)

**หน้าที่:** จัดการเวอร์ชันเอกสาร

**ข้อมูลที่จัดการ:**
- เวอร์ชัน (v1, v2, v3, ...)
- เอกสาร (Document ID, ชื่อเอกสาร)
- ผู้สร้างเวอร์ชัน
- วันที่สร้าง
- ขนาดไฟล์
- หมายเหตุการเปลี่ยนแปลง (Change Note)
- เวอร์ชันปัจจุบัน

**ฟังก์ชันหลัก:**
- ดูเวอร์ชันทั้งหมดของเอกสาร (จัดกลุ่มตามเอกสาร)
- ดาวน์โหลดเวอร์ชัน
- ย้อนกลับไปใช้เวอร์ชันเก่า (Restore)
- เปรียบเทียบเวอร์ชัน (เปรียบเทียบ 2 เวอร์ชัน)

**Flow การทำงาน:**
1. ดูเวอร์ชัน: แสดงเวอร์ชันทั้งหมดของเอกสาร (เรียงจากล่าสุดไปเก่า)
2. ดาวน์โหลด: คลิก "ดาวน์โหลด" → ดาวน์โหลดเวอร์ชันนั้น
3. ย้อนกลับ: คลิก "ย้อนกลับ" → ยืนยัน → ย้อนกลับไปใช้เวอร์ชันเก่า
4. เปรียบเทียบ: คลิก "เปรียบเทียบ" → เลือก 2 เวอร์ชัน → แสดงความแตกต่าง

**เงื่อนไขสำคัญ:**
- ระบบเก็บเวอร์ชันอัตโนมัติเมื่อมีการแก้ไข
- สามารถย้อนกลับไปใช้เวอร์ชันเก่าได้
- สามารถเปรียบเทียบเวอร์ชันได้

### 8. Permissions (สิทธิ์)

**หน้าที่:** กำหนดสิทธิ์การเข้าถึงเอกสาร (R/W/U/D)

**ข้อมูลที่จัดการ:**
- ประเภทสิทธิ์ (ตามเอกสาร, ตามหมวดหมู่, ตามผู้ใช้, ตามบทบาท)
- เอกสาร/หมวดหมู่/ผู้ใช้/บทบาท
- สิทธิ์การเข้าถึง:
  - Read (R) - อ่านได้
  - Write (W) - เขียน/สร้างได้
  - Update (U) - แก้ไขได้
  - Delete (D) - ลบได้

**ฟังก์ชันหลัก:**
- เพิ่มสิทธิ์ใหม่ (เลือกประเภท, เลือกเอกสาร/หมวดหมู่/ผู้ใช้/บทบาท, กำหนดสิทธิ์ R/W/U/D)
- แก้ไขสิทธิ์
- ลบสิทธิ์
- กรองตามประเภท (ทั้งหมด, ตามเอกสาร, ตามหมวดหมู่, ตามผู้ใช้, ตามบทบาท)

**Flow การทำงาน:**
1. เพิ่มสิทธิ์:
   - เลือกประเภทสิทธิ์ (ตามเอกสาร/หมวดหมู่/ผู้ใช้/บทบาท)
   - เลือกเอกสาร/หมวดหมู่/ผู้ใช้/บทบาท
   - กำหนดสิทธิ์ R/W/U/D
   - บันทึก
2. แก้ไข: คลิก "แก้ไข" → แก้ไขสิทธิ์ → บันทึก
3. ลบ: คลิก "ลบ" → ยืนยัน → ลบสิทธิ์

**เงื่อนไขสำคัญ:**
- สิทธิ์สามารถกำหนดได้ทั้งระดับเอกสาร, หมวดหมู่, ผู้ใช้, และบทบาท
- สิทธิ์ R/W/U/D เป็นอิสระต่อกัน

### 9. Templates (เทมเพลต)

**หน้าที่:** จัดการและเก็บไฟล์เทมเพลตเอกสารต่างๆ

**ข้อมูลที่จัดการ:**
- ชื่อเทมเพลต
- คำอธิบาย
- หมวดหมู่
- แผนก
- ไฟล์ (ชื่อไฟล์, ประเภทไฟล์, ขนาดไฟล์)
- วันที่สร้าง
- ผู้สร้าง

**ฟังก์ชันหลัก:**
- อัปโหลดเทมเพลตใหม่ (กรอกชื่อ, คำอธิบาย, เลือกหมวดหมู่, แผนก, อัปโหลดไฟล์)
- ดูเทมเพลต
- ดาวน์โหลดเทมเพลต
- ลบเทมเพลต
- ค้นหาและกรอง (ตามชื่อ, คำอธิบาย, หมวดหมู่, แผนก)

**Flow การทำงาน:**
1. อัปโหลดเทมเพลต:
   - กรอกชื่อเทมเพลต, คำอธิบาย
   - เลือกหมวดหมู่, แผนก (ไม่บังคับ)
   - อัปโหลดไฟล์ (รองรับ PDF, DOC, DOCX, XLS, XLSX)
   - บันทึก
2. ดู/ดาวน์โหลด: คลิก "ดู" หรือ "ดาวน์โหลด" → ดำเนินการตามต้องการ
3. ลบ: คลิก "ลบ" → ยืนยัน → ลบเทมเพลต

**เงื่อนไขสำคัญ:**
- รองรับไฟล์: PDF, DOC, DOCX, XLS, XLSX
- เทมเพลตใช้เป็นต้นแบบในการสร้างเอกสารใหม่

### 10. Settings (ตั้งค่า)

**หน้าที่:** จัดการการตั้งค่าแจ้งเตือนและการบูรณาการ

**ข้อมูลที่จัดการ:**
- การตั้งค่าแจ้งเตือน:
  - แจ้งเตือนเอกสารใกล้หมดอายุ (วันล่วงหน้า: 30, 15, 7 วัน หรือกำหนดเอง)
  - ช่องทางการแจ้งเตือน (Email, Line, ในระบบ)
  - แจ้งเตือนเฉพาะหมวดหมู่ (เว้นว่าง = ทุกหมวดหมู่)
  - แจ้งเตือนการอนุมัติ (เปิด/ปิด, ช่องทาง)
- ตั้งค่า Email:
  - เปิด/ปิด Email
  - Email Address
  - Subject
- ตั้งค่า Line:
  - เปิด/ปิด Line
  - Line Token
- Calendar Integration:
  - เปิด/ปิด Calendar Integration
  - จำนวนวันที่จะซิงค์ล่วงหน้า

**ฟังก์ชันหลัก:**
- กำหนดวันแจ้งเตือน (30, 15, 7 วัน หรือกำหนดเอง)
- เลือกช่องทางการแจ้งเตือน (Email, Line, ในระบบ)
- เลือกหมวดหมู่ที่ต้องการแจ้งเตือน
- ตั้งค่า Email (เปิด/ปิด, Email Address, Subject)
- ตั้งค่า Line (เปิด/ปิด, Line Token)
- ตั้งค่า Calendar Integration (เปิด/ปิด, จำนวนวันที่จะซิงค์ล่วงหน้า)
- บันทึกการตั้งค่า

**Flow การทำงาน:**
1. กำหนดวันแจ้งเตือน: เลือกวัน (30, 15, 7) หรือเพิ่มวันเอง → เพิ่ม
2. เลือกช่องทาง: เลือก Email, Line, ในระบบ (สามารถเลือกหลายช่องทาง)
3. เลือกหมวดหมู่: เลือกหมวดหมู่ที่ต้องการแจ้งเตือน (เว้นว่าง = ทุกหมวดหมู่)
4. ตั้งค่า Email/Line/Calendar: เปิด/ปิด → กรอกข้อมูล (ถ้าเปิด) → บันทึก

**เงื่อนไขสำคัญ:**
- สามารถกำหนดวันแจ้งเตือนได้หลายวัน
- สามารถเลือกช่องทางการแจ้งเตือนได้หลายช่องทาง
- Calendar Integration รองรับการ Export เป็น iCal/Google Calendar

### 11. Reports (รายงาน)

**หน้าที่:** รายงานและส่งออกข้อมูลเอกสาร

**ข้อมูลที่แสดง:**
- สรุปสถิติ:
  - เอกสารทั้งหมด
  - ขนาดไฟล์รวม
  - ใกล้หมดอายุ (ภายใน 30 วัน)
  - ค่าต่ออายุ (ภายใน 30 วัน)
- เอกสารแยกตามหมวดหมู่ (จำนวน, ขนาดไฟล์รวม)
- เอกสารแยกตามสถานะ (Active, Archived, Draft, Pending)
- เอกสารแยกตามระดับความลับ (Public, Internal, Confidential)
- กราฟวิเคราะห์ (จำนวนเอกสารตามหมวดหมู่, สถานะเอกสาร)

**ฟังก์ชันหลัก:**
- ดูรายงานสรุป
- ส่งออกรายงาน (Excel, PDF, CSV, Google Sheets):
  - รายงานเอกสารทั้งหมด
  - รายงานเอกสารใกล้หมดอายุ
  - รายงานค่าต่ออายุ
  - รายงานสรุปทั้งหมด

**Flow การทำงาน:**
1. ดูรายงาน: แสดงสรุปสถิติ, ตาราง, และกราฟ
2. ส่งออก: คลิก "ส่งออก Excel" หรือเลือกรายงาน → เลือกรูปแบบ (Excel/PDF/CSV/Google Sheets) → ส่งออก

## Flow การทำงานหลักของระบบ

### 1. การอัปโหลดเอกสารใหม่
1. ผู้ใช้อัปโหลดเอกสาร (กรอกข้อมูล, เลือกหมวดหมู่, อัปโหลดไฟล์)
2. กำหนดระดับความลับ, วันหมดอายุ (ถ้ามี), Tags
3. บันทึก → สถานะเป็น Draft หรือ Pending (รออนุมัติ)
4. ระบบสร้าง Audit Trail (Created)
5. ถ้าต้องการอนุมัติ → ส่งไปยัง Workflow การอนุมัติ

### 2. การอนุมัติเอกสาร
1. เอกสารถูกส่งไปยัง Workflow การอนุมัติ
2. ผู้อนุมัติในแต่ละขั้นตอนพิจารณา
3. อนุมัติ/ไม่อนุมัติ (พร้อมหรือไม่พร้อมลายเซ็น)
4. ระบบบันทึก Audit Trail (Approved/Rejected)
5. ถ้าอนุมัติครบทุกขั้นตอน → สถานะเป็น Active

### 3. การแก้ไขเอกสาร
1. ผู้ใช้แก้ไขเอกสาร
2. อัปโหลดไฟล์ใหม่
3. ระบบสร้างเวอร์ชันใหม่อัตโนมัติ
4. ระบบบันทึก Audit Trail (Updated)
5. สถานะอาจเปลี่ยนเป็น Pending (ถ้าต้องการอนุมัติอีกครั้ง)

### 4. การติดตามเอกสารใกล้หมดอายุ
1. ระบบตรวจสอบเอกสารที่มีวันหมดอายุ
2. คำนวณจำนวนวันที่เหลือ
3. แจ้งเตือนตามการตั้งค่า (Email, Line, ในระบบ)
4. แสดงใน Dashboard และหน้า Expiring
5. Export เป็น Calendar (iCal/Google Calendar)

### 5. การจัดการสิทธิ์
1. ผู้ดูแลระบบกำหนดสิทธิ์ (ตามเอกสาร/หมวดหมู่/ผู้ใช้/บทบาท)
2. กำหนดสิทธิ์ R/W/U/D
3. ระบบตรวจสอบสิทธิ์เมื่อมีการเข้าถึงเอกสาร
4. บันทึก Audit Trail (Viewed, Downloaded)
